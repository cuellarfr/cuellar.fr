How I work

Every project is different and I often wear different hats, but my fundamental approach to UX Design is consistent. It's a collaborative and iterative process where I work closely with product managers, other designers and developers in different stages. I enjoy this approach because it's effective and I'm always learning new things. These are the steps I follow in my design process.

1. Research

As a designer, my main job is to solve problems so I need to clearly define them. In order to do that, I interview stakeholders so I can understand the product vision and constraints. That also helps me define the project goals and schedule. After that, I normally review any existing work or product, and then I do a bit of competitive analysis in order to learn how other companies solved similar problems. Later, I do field studies and user interviews so I can understand their workflows, pain points, behaviors, needs, and map out use cases and scenarios.

2. Ideate

At this stage of the process, I start looking for solutions to the problem. I normally work with product managers on a story map and then I immediately start sketching and creating quick wireframes that I iterate so I can explore different UI solutions.

3. Prototype

Depending on the project, these wireframes can evolve into interactive prototypes that I build using tools like Sketch, Figma, Framer or Middleman.

4. User Test

Once I have a working prototype I need to start testing and refining it. I normally start with casual testing in the office where I ask some coworkers to complete tasks in order to find the most obvious usability issues. After that, I prepare some usability testing sessions with real users or users with a profile similar to our personas. I typically prefer in-person usability testing because it allows me to interact with users and ask follow-up questions. However, this can be time-consuming so sometimes I do remote testing using services like UserTesting. Once I analyze and extract the test results, I use this feedback to improve the prototype and once it's done, I test it again. The process is repeated until I reach an optimal solution for both users and stakeholders.

5. Implement

Lastly, I work with visual designers and front-end engineers during the final implementation of the solution. The result is often deployed for a few users using A/B testing. I use web analytics and user feedback in the last round of refinement of the product.
